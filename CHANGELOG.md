# История изменений

### 1.1.42 -- 2026-06-19
- **Кнопка обновления показывает обе версии** — «Обновить vX до vY» (текущая → доступная), а не только целевую

### 1.1.41 -- 2026-06-19
- **В подвале добавлена атрибуция форка** — «mod by DCRM» (ссылка на репозиторий) рядом с «© r0otx»

### 1.1.40 -- 2026-06-19
- **Apply докачивает только недостающее, а не весь набор.** Раньше добавление одного GeoIP-сервиса перекачивало все списки и заново тянуло большую базу GeoSite (v2fly, ~3.4 МБ), даже если она не менялась. Теперь скачивается только реально отсутствующее (например, один добавленный сервис), база GeoSite — лишь если её нет на диске. Update Now / авто-обновление по-прежнему делают полную пересборку

### 1.1.39 -- 2026-06-19
- **Шапка упрощена:** вместо «версия + обновлений нет + кнопка проверки» теперь одна кнопка — **«Текущая версия — vX»** (или **«Обновить до vX»**, если есть обновление). Клик открывает окно
- **Проверка и статус — в окне:** там кнопка **«Проверить обновления»**, строка состояния («обновлений нет» / «доступно обновление: vX» / «не удалось проверить») и кнопка «Обновить» (когда есть куда обновляться)

### 1.1.38 -- 2026-06-18
- **Бадж GitHub в шапке** рядом с Telegram — показывает последний релиз (shields.io) и ведёт на репозиторий. Если картинка не загрузилась (например, GitHub заблокирован) — автоматически заменяется текстовой ссылкой «🐙 GitHub»

### 1.1.37 -- 2026-06-18
- **Опция «Wipe before update»** (по умолчанию ВЫКЛ) для Update Now / авто-обновления. Выкл — старые списки сохраняются, если перекачка не удалась (безопаснее, рекомендуется); вкл — полная очистка перед загрузкой (гарантированно чистый набор, но при сбое скачивания список пропадёт). Раньше очистка была всегда, и при недоступности GitHub гео могло пропасть

### 1.1.36 -- 2026-06-18
- **«Update Now» и авто-обновление списков (cron) делают полную пересборку**: сначала удаляются все гео-файлы (на случай оставшегося мусора), затем весь набор скачивается заново и пересобираются правила. Apply / Force Apply по-прежнему инкрементальны (докачать добавленное, удалить снятое)

### 1.1.35 -- 2026-06-18
- **Удаление сервиса из поля GeoIP убирает и его файл списка.** Раньше при удалении сервиса из «GeoIP Service Lists» маршрут переставал применяться, но `.cidr`-файл оставался на диске. Теперь лишние файлы удаляются при Apply / Force Apply / Update / Start (GeoSite-категории и custom-домены/IP так чистились уже раньше)

### 1.1.34 -- 2026-06-18
- **Гео-списки автоматически докачиваются**, если настроены, но отсутствуют на диске: после обновления (пакет стирает `/opt/amneziawg`) и после Apply / Force Apply, когда добавил сервис/категорию, которых ещё нет в файлах. Загрузка идёт в фоне, прогресс виден в логе. Раньше после обновления гео слетало, а добавленные сервисы не качались до ручного «Update Now»
- **Подсказка GeoIP:** предупреждение «для youtube/discord/microsoft/… IP-списков нет — используйте GeoSite» теперь **жёлтым** (видно сразу); убрана невидимая строка «Works without DNS…»

### 1.1.33 -- 2026-06-18
- **Клик по версии в шапке открывает окно со списком изменений** (раньше вёл просто на репозиторий)
- **Когда обновлений нет** — вместо кнопки обновления выводится «обновлений нет» и кнопка **«Проверить обновления»** (ручная повторная проверка); при ошибке проверки — кнопка «Проверить» рядом с сообщением
- Окно changelog показывает список изменений нужной версии (на которую обновляешься, либо текущей) и прячет кнопку «Обновить», когда обновляться не на что

### 1.1.32 -- 2026-06-18
- **`S99amneziawg update`** — init-скрипт теперь принимает параметр `update`, запускающий весь каскад обновления (скачивание нового пакета через зеркала + проверка SHA256, `opkg install`, переустановка страницы, обновление статуса) — то же, что кнопка обновления в UI, но из консоли

### 1.1.31 -- 2026-06-18
- **Сообщение «⚠ не удалось проверить обновления»** в шапке, когда проверка новой версии на GitHub не прошла (`api.github.com` недоступен или превышен лимит запросов). Текущая версия при этом всё равно отображается; раньше ошибка была молчаливой

### 1.1.30 -- 2026-06-18
- **Версия в шапке снова всегда отображается** (со ссылкой на репозиторий). Раньше номер брался только при успешной проверке обновлений на GitHub — если `api.github.com` не отвечал (лимит запросов на стороне браузера или блокировка), версия пропадала. Теперь она берётся из локального статуса при каждом опросе, независимо от GitHub (кнопка «Update to …» по-прежнему зависит от проверки)

### 1.1.29 -- 2026-06-18
- **Убрано окно браузера «Повторить отправку формы»** (resubmit), всплывавшее через некоторое время после Apply / Update list / обновления. Причина: страница перезагружалась через `reload()` после POST-формы, и браузер просил повторить отправку. Теперь Apply и Update list **не перезагружают страницу** (статус и лог и так обновляются в реальном времени), а обновление прошивки перезагружает чистым GET

### 1.1.28 -- 2026-06-18
- **Исправлено описание поля «GeoIP Service Lists»** в UI. Раньше там были перечислены несуществующие сервисы (`apple, amazon, microsoft, github, stripe, openai`), которых нет в Loyalsoldier/geoip (отдают 404). Теперь указаны реально доступные: `telegram, google, facebook, twitter, netflix, cloudflare, fastly, cloudfront, tor` + коды стран (`us, ru, cn, …`). Для остальных (youtube, discord, microsoft, github и т.п.) IP-списков нет — нужно использовать GeoSite

### 1.1.27 -- 2026-06-18
- **Откат изменения geo-ipset из 1.1.23.** Записи доменов (GeoSite / custom-домены) снова истекают (timeout 24 ч), а не хранятся вечно: для доменов с часто меняющимися IP (CDN) постоянное хранение копило устаревшие адреса, и они ошибочно гнались через VPN. dnsmasq обновляет запись при каждом резолве, так что активные домены остаются, а неиспользуемые уходят. Статические списки GeoIP и custom-IP по-прежнему постоянны. Спасибо Artem Shlyakhtin за замечание

### 1.1.26 -- 2026-06-18
- **Окно со списком изменений больше не «залипает» на старой версии** — changelog подтягивается по версионному тегу (`@vX.Y.Z`) напрямую из репозитория, а не из закэшированного jsDelivr `@main`. Логика changelog на стороне роутера убрана — только запрос с фронтенда (jsDelivr → raw.githubusercontent)

### 1.1.25 -- 2026-06-18
- **Apply / Restart / Force Apply больше не «подвешивают» обработку событий на 60-90 секунд** — проверка связи туннеля (health-check) вынесена в фон, обработчик события завершается сразу. Раньше rc_service всё это время был занят и молча пропускал другие события (Stop, повторный Apply и системные)

### 1.1.24 -- 2026-06-18
- **Атомарная запись статус-файла** — UI больше не может прочитать «рваный» JSON
- **Обновление: при провале `opkg install`** теперь сообщается об ошибке и остаётся рабочая версия (раньше всегда писалось «complete»)
- **Безопасность: пакет из зеркала без SHA256 отклоняется** (прямую загрузку с github.com по TLS по-прежнему пропускаем)
- **Валидация S3/S4** (как у S1/S2) и строже проверка H1-H4 — мусор вроде `-`/`--` отсекается, диапазоны `5-10` сохранены
- **Приватный ключ и конфиг создаются с `umask 077`** (не world-readable)
- **GeoIP: «битый» список из HTML-ответа зеркала отбрасывается** (проверка на наличие IPv4-строк)
- **GeoSite-извлечение быстрее** — awk больше не дочитывает весь файл базы на каждую категорию
- **Полнее чистка при удалении** (changelog/update/log в /www/user) и хуки `/jffs/scripts/*` всегда получают +x; include из dnsmasq.conf.add удаляется по точному совпадению

### 1.1.23 -- 2026-06-18
- **Apply больше не рвёт все соединения роутера** — `flush_conntrack` сбрасывает только помеченные записи (раньше при отсутствии помеченных потоков делал полный сброс conntrack всей сети, рвя чужие загрузки/SSH)
- **GeoSite и custom-домены не отваливаются через 24 ч** — записи geo-ipset больше не истекают (timeout 0)
- **Защита от IPv6 DNS-leak (filter-AAAA) применяется и для vpn_all** — раньше dnsmasq не перечитывался, если нет geo/доменов
- **GeoIP-сервисы регистронезависимы** — `Telegram` с заглавной больше не ломает загрузку списка
- **DNS-перехват: защита от дубликатов (`-C`) и от «осиротевшего» правила** при смене IP роутера
- **Watchdog: backoff при недоступном сервере** (не штормит каждые 5 мин) + проверка mangle-хука, а не только маршрута

### 1.1.22 -- 2026-06-18
- **Логи на странице — в реальном времени** и с первого действия (Apply/Update/Start/Stop), включая фоновые шаги (перезапуск dnsmasq). Больше не нужно лезть в системный лог, чтобы понять, отработал ли restart. По заявке Artem Shlyakhtin
- **Watchdog (само-восстановление маршрутов) больше не теряется** после Apply / firewall-restart / обновления гео — раньше он молча удалялся и не восстанавливался, отключая защиту до перезагрузки
- **Маршрут трафика самого роутера (prio 100) восстанавливается** при Apply; устранён бесконтрольный рост `ip rule` (правила prio 97 для direct-устройств теперь чистятся)
- **Custom-домены и custom-IP больше не стираются** при сохранении без скачанной GeoIP-базы (они и не требуют скачивания)
- **`Stop` надёжно перезапускает dnsmasq** — раньше при остановке у всей сети могло остаться подавление IPv6 и битые ipset-правила
- Перезапуски dnsmasq сериализованы (без «пинг-понга» при совпадающих операциях)

### 1.1.21 -- 2026-06-18
- **GeoSite-домены применяются сразу после Apply / Update list** — раньше перезапуск dnsmasq, вызванный из обработчика события, прошивка пропускала («skip the event: restart_dnsmasq»), и домены не работали до ручного `service restart_dnsmasq`. Теперь перезапуск откладывается и повторяется, пока не сработает
- **В лог пишется конкретный URL, который не удалось скачать** (GeoIP/GeoSite) — видно, какой список недоступен (например, несуществующий сервис → 404)

### 1.1.20 -- 2026-06-18
- **При загрузке страницы статус показывает «Загрузка…»**, а не «Stopped» — раньше, пока данные ещё не пришли, бейдж выглядел как «остановлено», хотя интерфейс только грузился

### 1.1.19 -- 2026-06-18

- **Удаление сервиса/категории убирает и маршруты**: при удалении из полей GeoIP / GeoSite / Custom Domains старые файлы больше не остаются и не попадают в правила

### 1.1.13 -- 2026-06-18
- **Установка и обновление работают через зеркала**, когда GitHub-CDN с релизами недоступен (частая ситуация в РФ): онлайн-установщик и встроенный апдейтер при неудаче пробуют зеркала и **проверяют SHA256** пакета (защита от подмены)

### 1.1.12 -- 2026-06-18

- **Версия в шапке** — теперь ссылка на репозиторий
- **Ссылка на Telegram-чат** в подвале интерфейса

### 1.1.11 -- 2026-06-18
- **Кнопка Force Apply** — сохраняет настройки и делает полный перезапуск VPN: применяет в т.ч. изменения WireGuard-параметров (ключи, endpoint, обфускация, MTU — чего обычный Apply не делает) и полностью пересобирает маршруты/firewall/гео
- **Исправлен запуск на роутерах, не подгружающих `tun` автоматически** (напр. RT-AC68U) — модуль `tun` теперь грузится надёжно (раньше `modprobe` не находился из-за PATH)

### 1.1.10 -- 2026-06-18

Раньше после перезапуска файрвола маршруты в таблице VPN могли пропадать, и GeoSite/туннель переставали работать до перезапуска. (диагностика — Artem Shlyakhtin)

- **Статус подключения** не мерцает при таймауте опроса и показывает «Connecting» в т.ч. после обновления страницы

### 1.1.8 -- 2026-06-18
